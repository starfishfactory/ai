# 에이전트 활용 가이드

에이전트 목록과 상세 설명은 [agents/README.md](../agents/README.md)를 참고하라.
이 문서는 효과적인 활용법과 팁을 다룬다.

## 효과적인 에이전트 활용법

### 구체적으로 요청하기

모호한 요청보다 구체적인 요청이 정확한 결과를 가져온다.

```text
# 모호한 요청
"코드를 확인해줘"
"테스트를 작성해줘"

# 구체적인 요청
"이 사용자 인증 함수의 보안과 성능을 검토해줘"
"결제 처리 함수의 TDD 테스트를 작성해줘. 실패 케이스와 엣지 케이스를 포함해서"
```

### 컨텍스트 제공하기

환경, 기술 스택, 증상을 함께 전달하면 더 정확한 응답을 얻는다.

```text
# 컨텍스트 부족
"에러가 발생했어"

# 컨텍스트 충분
"Node.js 환경에서 PostgreSQL 연결 시 다음 에러가 발생한다: [에러 메시지]"
```

### 단계별로 진행하기

하나의 대화에서 모든 것을 처리하기보다, 단계별로 나누어 진행하는 것이 효과적이다.

**Core Pack 워크플로우:**

```text
1단계: 테스트 작성 (test-generator)
2단계: 코드 구현
3단계: 리뷰 (code-reviewer)
4단계: 개선 및 완료
```

**Advanced Pack 워크플로우:**

```text
1단계: API 설계/문서화 (dfs)
2단계: TDD 개발 (test-generator)
3단계: 보안 강화 (security-auditor)
4단계: 최종 리뷰 (code-reviewer)
```

## 활용 시나리오

### 기능 개발 워크플로우

새로운 기능을 구현할 때 테스트 주도로 진행하는 흐름이다.

```text
1. test-generator로 테스트 케이스 먼저 작성
2. 테스트를 통과하도록 기능 구현
3. code-reviewer로 코드 품질 검토
4. 피드백 반영 후 완료
```

### API 문서화 워크플로우

API의 전체 플로우를 분석하고 문서화하는 흐름이다.

```text
1. dfs로 API 기능명세 작성
2. korean-docs로 사용자 가이드 작성
```

### 배포 전 검증 워크플로우

배포 전에 보안과 코드 품질을 최종 점검하는 흐름이다.

```text
1. security-auditor로 OWASP 기반 보안 검토
2. code-reviewer로 코드 품질 최종 확인
```

## 팁

- **자동 선택**: 작업의 성격을 명확히 표현하면 적절한 에이전트가 자동으로 선택된다.
- **명시적 호출**: `@에이전트명`으로 직접 지정할 수 있다.
- **자동 선택 확률**: 에이전트의 `description`에 PROACTIVELY 키워드가 포함되면 자동 선택 확률이 높아진다.
- **문서화 먼저**: 큰 그림을 그린 뒤 구현에 들어가는 것이 효율적이다.
- **점진적 개선**: 작동 확인 후 최적화, 보안 순으로 진행한다.

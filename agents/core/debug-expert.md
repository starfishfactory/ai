체계적인 디버깅 프로세스로 문제를 해결합니다.

디버깅 단계:
1. 문제 재현: 에러를 일관되게 재현
2. 정보 수집: 로그, 스택 트레이스, 환경 정보 분석
3. 가설 수립: 근본 원인에 대한 가설 설정
4. 검증: 가설을 테스트하여 원인 확인
5. 해결: 문제 수정 및 테스트
6. 예방: 재발 방지를 위한 개선 사항 제안

로그 분석, 스택 트레이스 해석, 재현 가능한 테스트 케이스 작성을 포함합니다.

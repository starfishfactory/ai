## 개요

이 저장소는 개인 프로젝트 개발에 최적화된 Claude Code 서브에이전트들을 모아놓은 라이브러리입니다. 각 에이전트는 다음과 같은 개인 개발 스타일을 반영하여 설계되었습니다:

- **🧪 TDD 중심**: 테스트 우선 개발 방식
- **🇰🇷 한국어 문서화**: 상세하고 체계적인 한국어 설명
- **✨ 깃모지 활용**: PR과 커밋에 깃모지 사용
- **📝 단계별 검증**: 각 단계마다 철저한 검증

## 에이전트 구성

### 🛠️ Core 에이전트 (범용)
| 에이전트 | 목적 | 주요 기능 |
|----------|------|-----------|
| `code-reviewer` | 코드 리뷰 | 품질, 보안, 성능 검토 |
| `test-generator` | 테스트 생성 | TDD 기반 테스트 케이스 작성 |
| `debug-expert` | 디버깅 | 체계적인 문제 해결 |
| `documentation-helper` | 문서화 | 한국어 기술 문서 작성 |

### 🎯 Personal 에이전트 (개인 특화)
| 에이전트 | 목적 | 개인화 요소 |
|----------|------|-------------|
| `tdd-coach` | TDD 가이드 | Red-Green-Refactor 사이클 |
| `korean-docs` | 한국어 문서 | 체계적 마크다운 구조 |
| `gitmoji-pr` | PR 작성 | 깃모지 기반 PR 템플릿 |
| `step-validator` | 단계별 검증 | 완료 조건 체크리스트 |

### 🚀 Specialized 에이전트 (전문 분야)
| 에이전트 | 목적 | 전문 분야 |
|----------|------|-----------|
| `api-architect` | API 설계 | REST API 아키텍처 |
| `performance-optimizer` | 성능 최적화 | 병목 분석 및 개선 |
| `security-auditor` | 보안 감사 | 취약점 분석 및 강화 |

## 설치 및 설정

### 1. 저장소 클론
```bash
# molidae 폴더에 클론 (이미 완료)
cd ~/molidae/ai/claude-code
```

### 2. 심볼릭 링크 설정

#### 전체 에이전트 링크 (추천)
```bash
# 사용자 레벨 (모든 프로젝트에서 사용)
ln -sf ~/molidae/ai/claude-code/agents/* ~/.claude/agents/

# 또는 프로젝트별 설정
ln -sf ~/molidae/ai/claude-code/agents/* ./.claude/agents/
```

#### 카테고리별 선택적 링크
```bash
# Core 에이전트만
ln -sf ~/molidae/ai/claude-code/agents/core/* ~/.claude/agents/

# Personal 에이전트 추가
ln -sf ~/molidae/ai/claude-code/agents/personal/* ~/.claude/agents/

# Specialized 에이전트 추가
ln -sf ~/molidae/ai/claude-code/agents/specialized/* ~/.claude/agents/
```

### 3. 설정 확인
```bash
# 설치된 에이전트 확인
ls -la ~/.claude/agents/

# Claude Code에서 확인
/agents
```

## 사용법

### 자동 에이전트 선택
Claude가 작업 내용을 분석하여 적절한 에이전트를 자동으로 선택합니다.

```text
# 예시 1: 코드 리뷰 요청
> 이 함수의 성능을 개선해주세요
→ code-reviewer 에이전트가 자동 선택

# 예시 2: TDD 개발
> 새로운 기능을 TDD로 개발하고 싶어요
→ tdd-coach 에이전트가 자동 선택

# 예시 3: 문서 작성
> 이 API에 대한 사용 가이드를 작성해주세요
→ korean-docs 에이전트가 자동 선택
```

### 명시적 에이전트 호출
특정 에이전트를 직접 지정할 수도 있습니다.

```text
# 특정 에이전트 지정
> step-validator 에이전트로 현재 단계를 검증해주세요
> gitmoji-pr 에이전트로 이 변경사항의 PR을 작성해주세요
```

## 에이전트 상세 설명

### 🧪 TDD Coach
- **목적**: TDD 방법론 가이드
- **특징**: Red-Green-Refactor 사이클 안내
- **사용 시기**: 새로운 기능 개발 시작 시
- **출력**: 단계별 TDD 가이드라인

### 🇰🇷 Korean Docs
- **목적**: 체계적인 한국어 문서 작성
- **특징**: 목차, 예시, 베스트 프랙티스 포함
- **사용 시기**: README, API 문서, 가이드 작성 시
- **출력**: 구조화된 마크다운 문서

### ✨ Gitmoji PR
- **목적**: 깃모지 기반 PR 작성
- **특징**: 변경사항에 맞는 깃모지 선택
- **사용 시기**: Pull Request 생성 시
- **출력**: 깃모지가 포함된 PR 제목과 설명

### 📝 Step Validator
- **목적**: 개발 단계별 검증
- **특징**: 완료 조건 체크리스트 제공
- **사용 시기**: 각 개발 단계 완료 후
- **출력**: 검증 체크리스트와 다음 단계 가이드

## 개인화 특징

### 🎯 TDD 중심 개발
- 모든 에이전트가 테스트 우선 접근법을 권장
- Red-Green-Refactor 사이클을 기본으로 가이드
- 테스트 케이스 작성을 우선으로 제안

### 🇰🇷 한국어 친화적
- 모든 설명과 피드백을 한국어로 제공
- 기술 용어의 한국어 설명 병행
- 한국 개발자에게 친숙한 예시 사용

### 📊 체계적 문서화
- 목차와 구조화된 내용
- 단계별 상세 설명
- 실제 활용 가능한 예시 포함

### ✅ 단계별 검증
- 각 단계마다 명확한 완료 조건
- 다음 단계 진행 전 검증 항목
- 품질 보증을 위한 체크포인트

## 활용 예시

### 새로운 기능 개발 워크플로우

1. **TDD 시작**
   ```text
   > 사용자 인증 기능을 TDD로 개발하고 싶어요
   ```
   → `tdd-coach`가 Red-Green-Refactor 사이클 가이드

2. **테스트 작성**
   ```text
   > 로그인 함수에 대한 테스트를 작성해주세요
   ```
   → `test-generator`가 포괄적인 테스트 케이스 생성

3. **코드 구현 후 리뷰**
   ```text
   > 이 로그인 로직을 검토해주세요
   ```
   → `code-reviewer`가 품질, 보안, 성능 검토

4. **단계별 검증**
   ```text
   > 현재 단계가 완료되었는지 확인해주세요
   ```
   → `step-validator`가 완료 조건 체크

5. **문서화**
   ```text
   > 인증 API 사용 가이드를 작성해주세요
   ```
   → `korean-docs`가 체계적인 문서 생성

6. **PR 생성**
   ```text
   > 인증 기능 추가에 대한 PR을 작성해주세요
   ```
   → `gitmoji-pr`이 깃모지 포함 PR 생성

### 성능 최적화 워크플로우

1. **성능 분석**
   ```text
   > 이 API의 성능을 분석하고 개선해주세요
   ```
   → `performance-optimizer`가 병목 분석

2. **보안 검토**
   ```text
   > 최적화된 코드의 보안을 점검해주세요
   ```
   → `security-auditor`가 보안 취약점 검사

3. **API 설계 검토**
   ```text
   > 이 API 구조가 적절한지 검토해주세요
   ```
   → `api-architect`가 아키텍처 분석

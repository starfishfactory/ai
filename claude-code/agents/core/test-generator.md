TDD 방식으로 테스트 케이스를 생성합니다.

Red-Green-Refactor 사이클에 따라:
1. Red: 실패하는 테스트를 먼저 작성
2. Green: 테스트를 통과하는 최소한의 코드 작성
3. Refactor: 코드를 개선하면서 테스트 유지

테스트는 다음을 포함합니다:
- 정상 케이스 (Happy path)
- 엣지 케이스 (Edge cases)
- 경계값 테스트 (Boundary tests)
- 에러 케이스 (Error handling)

모든 테스트 코드는 한국어 주석과 함께 제공됩니다.

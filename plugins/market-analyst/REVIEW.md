# Market Analyst 플러그인 비관적 평가

**작성일**: 2026-01-08
**상태**: 초기 구현 완료, 검증 필요

---

## 1. 구조적 문제점

### 1.1 에이전트-명령어 중복

| 파일 | 문제 |
|------|------|
| `commands/pestel.md` | 전체 분석 프로세스 포함 |
| `agents/macro-analyst.md` | 거의 동일한 내용 중복 |

**영향**:
- 명령어에 이미 전체 분석 프로세스가 기술됨
- 에이전트 파일이 실질적으로 불필요한 중복 가능성
- Claude Code가 에이전트를 어떻게 호출하는지 **실제 동작 메커니즘 미검증**

### 1.2 에이전트 연계 미구현

```markdown
## 연계 에이전트
- **→ strategic-analyst**: O/T 분석 결과 전달
```

**문제점**:
- "전달"이라고 명시했지만 실제 연계 로직 없음
- 에이전트 간 데이터 전달 방법 미정의
- `/full-analysis`에서 순차 호출 방법 불명확
- 실제 작동 여부 테스트 안 함

---

## 2. 실용성 문제

### 2.1 데이터 소스 부재

| 항목 | 현재 상태 | 필요한 것 |
|------|----------|----------|
| 실시간 데이터 | ❌ 없음 | GDP, 환율, 시장점유율 등 |
| MCP 연동 | ⚠️ 기본만 | 실제 데이터 API 필요 |
| 데이터 정확성 | ❌ 보장 없음 | 구조화된 데이터 소스 |

**현재 `.mcp.json` 한계**:
```json
{
  "fetch": { ... },      // 일반 웹 페이지만 가능
  "filesystem": { ... }  // 로컬 파일 저장만 가능
}
```

**필요하지만 없는 것**:
- 금융 데이터 API (Yahoo Finance, Alpha Vantage)
- 산업 보고서 API (Statista, IBISWorld)
- 뉴스 API (NewsAPI)

### 2.2 출력 품질 불확실

```markdown
### 품질 기준
- 모든 6개 요인 분석 필수
- 신뢰할 수 있는 출처 명시
```

**문제점**:
- 품질 기준을 문서화했으나 강제할 방법 없음
- Claude가 기준을 따를지 보장 없음
- 출처 검증 메커니즘 없음
- 할루시네이션 방지 장치 없음

## 3. 사용자 경험 문제

### 3.1 긴 출력

예상 `/pestel 전기차 산업` 출력: **200-300줄**

- 터미널에서 읽기 어려움
- 스크롤 필요
- 핵심 인사이트 파악 어려움

### 3.2 인터랙션 부재

```bash
/pestel 전기차 산업
# → 긴 보고서 한 번에 출력
# → 끝. 추가 질문이나 수정 불가
```

**부재 기능**:
- 분석 범위 확인 프롬프트
- 중간 결과 확인
- 점진적 분석

### 3.3 한국어/영어 혼용

```markdown
| 요인 | 현황 | 트렌드 | 영향도 |
|------|------|--------|--------|
| Political | | ↑/→/↓ | 상/중/하 |
```

- 컬럼명은 한국어, 요인명은 영어
- 일관성 부족

## 4. 기술적 한계

### 4.1 Claude Code 플러그인 시스템 이해 부족

```json
{
  "commands": "./commands/",
  "agents": "./agents/",
  "skills": "./skills/"
}
```

**검증 필요 사항**:
| 질문 | 상태 |
|------|------|
| `agents/` 파일들이 실제 서브에이전트로 호출되는가? | ❓ 미검증 |
| `skills/` 파일들이 자동으로 컨텍스트에 로드되는가? | ❓ 미검증 |
| 명령어에서 "스킬 참조"가 실제로 작동하는가? | ❓ 미검증 |

### 4.2 allowed-tools 제한

```yaml
allowed-tools: WebSearch, WebFetch, Read, Write
```

**문제점**:
- `Task` 도구 미포함 → 에이전트 호출 불가능 가능성
- `Bash` 미포함 → 외부 스크립트 실행 불가

## 5. 콘텐츠 문제

### 5.1 스킬 파일 장황함

`skills/porter-five-forces.md`: **200줄+**

- 이론 설명이 과도함
- Claude는 이미 Porter를 알고 있음
- 컨텍스트 낭비 가능성

### 5.2 실제 예시 부재

README의 예시:
```markdown
/pestel 전기차 산업
**출력**: 정치적, 경제적, 사회적...
```

**문제점**:
- 실제 출력 예시 없음
- 사용자가 기대할 결과를 알 수 없음

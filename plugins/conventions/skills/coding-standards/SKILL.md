## Quick Rules

1. **Naming**: Descriptive, consistent with language conventions
2. **Error handling**: Explicit, typed errors at boundaries
3. **Testing**: Unit tests required for business logic
4. **Documentation**: Code comments in English, user docs in Korean
5. **Dependencies**: Minimize external dependencies, pin versions

Read `references/` files for language-specific details.

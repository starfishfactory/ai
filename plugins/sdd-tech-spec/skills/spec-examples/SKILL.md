# Tech Spec Writing Pattern Examples

Reference examples of common patterns in Tech Spec writing.

## Pattern Categories

| Category | Key Lesson |
|----------|-----------|
| Goals & Non-Goals | Specific + measurable > vague adjectives. Min 3 goals, 2 non-goals |
| NFR Concretization | "High availability" → "99.9% uptime (≤8.76h/year)" |
| Risk Matrix | Classify type (tech/schedule/external), assess probability + impact, provide mitigation |
| Mermaid Diagrams | Sequence, Architecture (graph TD), State diagrams |
| Alternative Review | Compare ≥2 alternatives with pros/cons, state selection rationale |
| Acceptance Criteria | Given/When/Then format with specific values |
